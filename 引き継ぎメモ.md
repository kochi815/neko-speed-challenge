# ねこバトルチャレンジ — 引き継ぎメモ

## プロジェクト概要
- 『ドラゴン最強王図鑑 バトルコロシアム（Switch）』をトレースし、算数の勉強と融合させた子ども向けバトルゲーム
- 対象: 小学校1年生～2年生前半（掛け算は含まない）
- HTML/CSS/JS のみ（フレームワークなし）、iPad/iPhone での使用を想定
- ひらがな表記にしなくてよい

## ゲームシステム
- **60ステージ × 6エリア**（各エリア10ステージ、最終ステージがボス戦）
- **2つのゲームモード**: 1年生（足し算・引き算）/ 4年生（四則演算＋穴埋め）
- **タイマー攻撃**: 回答が遅いと敵が自動攻撃（ステージ進行で加速）
- **HP持ち越し**: エリア内のステージ間でHPを引き継ぐ
- **必殺技ゲージ**: 回答評価に応じてゲージ蓄積、MAX時に敵最大HPの30%ダメージ
- **アイテムシステム**: やくそう（HP+8）/ バリア（1回防御）/ スピードブーツ（時間余裕）、最大3個所持
- **にゃんたろう進化**: 6段階の進化（ステージ進行で自動進化）
- **エリアマップ**: ほのおのどうくつ→あらしのさんみゃく→しんかいのしんでん→おうごんのめいきゅう→てんくうのしろ→あんこくのぎょくざ
- **トレーニングモード**: 8種類の問題タイプ、50秒タイムアタック
- **エリア別BGM**: フォールバック対応

## 修正済みバグ一覧（計10件）

### Bug 1: startBtn の二重ハンドラ
- **場所**: script.js 末尾（イベント登録部分）
- **内容**: `addEventListener` と `onclick` の両方が登録され、ボタン押下時に2回実行される
- **修正**: `addEventListener` を削除し `onclick` のみに統一

### Bug 2-3: 画面遷移時にオーバーレイが残る
- **場所**: script.js `hideAllScreens()`
- **内容**: ボス撃破オーバーレイや進化オーバーレイが画面遷移後も表示され続ける
- **修正**: `hideAllScreens()` 内で `bossDefeatedOverlay` と `evolutionOverlay` も非表示にする処理を追加

### Bug 4: 不正解ダメージ後のHP低下警告音が鳴らない
- **場所**: script.js `handleAnswer()` 内、不正解処理後
- **内容**: 正解時のダメージ計算部分にのみHP低下チェックがあり、不正解でHPが減った場合に警告音が鳴らない
- **修正**: 不正解ダメージ処理後にも `playSound('hpLow')` チェックを追加

### Bug 5: バリアの二重使用
- **場所**: script.js `useItem()` の barrier case
- **内容**: バリア有効中にさらにバリアを使うとアイテムが消費されるが効果が重複しない
- **修正**: `hasBarrier` が true の場合は早期 return してアイテムを消費しない

### Bug 6: 音声クローンのメモリリーク
- **場所**: sound.js `playSE()` のcloneNode処理
- **内容**: `cloneNode` で生成した Audio 要素の `src` が再生後もメモリに残り続ける
- **修正**: `ended` イベントと `catch` で `clone.src = ''` を実行しGC対象にする

### Bug 7: 全クリア後の「つづきから」ボタン表示
- **場所**: script.js スタート方法選択画面のボタン制御
- **内容**: 全60ステージクリア後に「つづきから」ボタンが Lv61 と表示され、存在しないステージを指す
- **修正**: `highestClearedStage >= maxStage` の場合に「Lv60 さいしゅうボス」と表示し最終ボス再戦を可能にする

### Bug 8: evolutionOverlay の z-index 競合
- **場所**: index.html の evolutionOverlay のインラインスタイル
- **内容**: インラインの `z-index: 50` がCSS の `z-index: 55` を上書きし、他の要素に隠れる
- **修正**: インラインスタイルから `z-index: 50;` を削除（CSSの z-index: 55 が有効に）

### Bug 9: バリア防御時にコンボがリセットされる
- **場所**: script.js `handleAnswer()` のバリア処理分岐
- **内容**: バリアで敵攻撃を防いだ際に不正解扱いのコンボリセットが走る
- **修正**: バリア成功時はコンボを維持し、専用の `barrierBlock` 効果音を再生

### Bug 10: バトル中断時に setTimeout がクリアされない（最終チェックで発見）
- **場所**: script.js `startBattle()`, `handleAnswer()`, `useSkill()` の setTimeout
- **内容**: `setTimeout(showQuestion, 1500)` 等が変数に保存されず、「やめる」ボタンでバトル中断後もバックグラウンドで showQuestion() が発火し、ゲーム状態が壊れる
- **修正**: `setGameTimeout()` / `clearAllGameTimeouts()` ラッパー関数を追加。3箇所の setTimeout を setGameTimeout に変更。「やめる」ボタンと `endBattle()` の両方で `clearAllGameTimeouts()` を呼び出し

## バランス調整（計4項目、Monte Carlo シミュレーション5000試行で検証済み）

### 1. 敵HPマルチプライヤー引き上げ
- **1年生モード**: 0.55 → 0.67（エリアごとに段階的に上昇）
- **4年生モード**: 0.66 → 0.88（エリアごとに段階的に上昇）
- ステージ進行に伴い適切に難易度が上がるように6段階で設定

### 2. 不正解時の敵HP回復率
- 0.03 → 0.05 に増加（ミスのペナルティを強化）

### 3. 必殺技ゲージ蓄積量の削減
- Critical: 25→20、Perfect: 15→12、Great: 10→8、Good: 5→4
- 必殺技の発動頻度を下げてゲームバランスを引き締め

### 4. ボス戦HP回復ボーナス
- ボスステージ開始時に HP+10 の回復を追加（HP持ち越しの過酷さを適度に緩和）
- マップ画面でもボス戦の回復量を反映した予想HPを表示
